1. Product architecture

Three-tier design: mobile clients (PWA / native), API gateway with RBAC, event-driven sync for offline queues. Media goes to object storage with signed URLs; location proofs are validated server-side against agency coordinates.

Client layer

Android-first app (Kotlin) or Capacitor PWA. Local SQLite/Realm for offline. Camera + EXIF strip + GPS from fused location provider.

Services

Auth (OTP), Users/Roles, Agencies, Visits, Expenses, Tasks, Notifications, Analytics jobs, Geo service (Haversine), Media ingest, Audit log.

Data

PostgreSQL (OLTP), Redis (sessions/rate limits), S3-compatible storage, optional ClickHouse/BigQuery for analytics.

Integrations

Maps (navigation deep links), WhatsApp Business API, future CRM webhooks, accounting export (CSV/Excel).

3. REST API structure (representative)

Version prefix /v1. JSON bodies. Pagination: cursor or page. Errors: RFC7807-style { code, message, details }.

4. Geo validation logic

Rules

Compute Haversine distance between visit (lat,lng) and agency (lat,lng). If agency has no geocode, flag pending until admin sets location. Tolerance default 150m (configurable 100–200m).

Flags & fraud signals

Visit Verified distance ≤ tolerance. Location Mismatch if > tolerance or accuracy > threshold (e.g. 50m). Dashboard: mock GPS, impossible speed between visits, duplicate photo hash.

9. Offline mode & sync

Envelope pattern

Each offline action gets client_id (UUID), created_at, and payload. Sync POST returns per-item accepted | conflict | duplicate. Visits stay “pending verification” until media uploads complete. Conflict: server wins on agency assignment; client wins on draft notes if not yet reviewed.

10. Advanced modules (roadmap hooks)

  • Lead scoring: rule engine + decay over inactivity; surfaces in agency list.
  • Route optimization: TSP-lite daily route from planned pins (Maps Directions API).
  • Territory mapping: manager draws regions; assignments auto-filter.
  • Duplicate detection: normalized phone + fuzzy name match + geo clustering.
  • AI insights: batch job flags inactive clients, suggests “next best action” push.
